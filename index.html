Builderius CSS framework

Builderius CSS is a utility-first framework designed to simplify and speed up web development. It provides a mix of atomic classes and pre-styled components, allowing for flexible and consistent design. The framework includes basic UI elements like buttons (filled, tonal, outlined, and ghost), badges, form fields, and accordions. Additionally, it offers custom components such as pagination and a modal-based cookie notice dialog. Built with scalability in mind, Builderius CSS helps developers create responsive and maintainable interfaces with ease.

# Colors

The Builderius CSS framework uses the modern OKLCH color space for perceptually uniform color generation. Each color is defined by a single --original CSS variable, from which all shades (50-950) are automatically calculated using CSS relative color syntax. This approach ensures consistent lightness steps and maintains color harmony across the entire palette. The framework intelligently adapts colors between light and dark themes, adjusting both lightness and chroma for optimal readability.

Accordion icon circle background

How does the icon circle background variable keep the circle aligned with the expand icon when the question text is very long and wraps across multiple lines?

The --accordion--icon-circle-color variable renders a filled circle behind the icon. Set it to any color on the <details> element — the circle is drawn on the summary's background, outside the masked pseudo-element, so it is not clipped by the icon mask.

Does the icon circle feature work correctly with both the default left-side icon position and the right-side icon position when the accordion label spans several lines of text?

Works with both default (left) and .icon-right icon positions.
